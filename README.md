# saibraman


## Описание 

Практический инструмент saibraman (Smart Artificial Intelligence for BRAin MANagement) для эффективного взаимодействия исследователя с искусственным интеллектом (ИИ). Общая идея: клиент следит за папками проектов пользователя, запросы пишутся в markdown файлах, (опциональный) сервер проксирует запросы в OpenRouter, ответ автоматически вставляется обратно в markdown файл.

> В текущей версии поддерживается взаимодействие с моделями ИИ только через интерфейс OpenRouter. Код реализован в логике асинхронной клиент-серверной архитектуры; взаимодействие происходит по схеме: клиент (персональный компьютер) <-> личный сервер пользователя <-> OpenRouter. При отсутствии реального личного сервера, он может быть запущен на персональном компьютере.


## Установка

1. Установить менеджер python-пакетов [anaconda](https://www.anaconda.com)

2. Создать виртуальное окружение:
    ```bash
    conda create --name sai python=3.12 -y
    ```

3. Активировать окружение:
    ```bash
    conda activate sai 
    ```

4. Установить зависимости:
    ```bash
    pip install aiohttp aiofiles python-docx requests
    ```

5. Клонировать репозиторий
    ```bash
    git clone https://github.com/AndreiChertkov/saibraman.git
    ```
    > И перейти в папку проекта для последующей работы.

6. Опционально удалить окружение по завершении работы с данным проектом:
    ```bash
    conda activate && conda remove --name sai --all -y
    ```

> Приведённый перечень действий нужно выполнить как на клиенте, так и на собственном сервере (если он используется).


## Настройка

1. Создать в папке `saibraman` файл конфигурации клиента `sai_config_client.toml`, используя существующий файл `sai_config_client_demo.toml` как образец

2. Создать в папке `saibraman` файл конфигурации сервера `sai_config_server.toml`, используя существующий файл `sai_config_server_demo.toml` как образец

3. Опционально можно задать в папке `server_prompts` промпты для пользователей сервера (приводятся в `txt` файлах, имеющих имя, совпадающее с именем существующего пользователя, заданным в `sai_config_server.toml`)

4. Опционально можно создать файл с заготовленными шаблонными фразами `sai_phrases.md`, используя существующий файл `sai_phrases_demo.md` как образец
    > Фразы задаются каждая в отдельной строке. Название фразы (например, `&&&-FIXOUT`) отделяется от соответствующего содержимого (подстановки) тремя стрелками с одинарным пробелом с обеих сторон. Заданные в файле фразы могут использоваться при любых запросах к ИИ (все вхождения имени фразы будут заменены на соответствующий фразе текст)
    > По умполчанию используется `sai_phrases_demo.md`. При использовании собственного файла, необходимо прописать соответствующий путь в файле конфигурации `sai_config.toml`

5. Опционально можно создать файл с заготовленными промптами `sai_prompts.md`, используя существующий файл `sai_prompts_demo.md` как образец
    > Заданные в файле промпты могут использоваться при любых запросах к ИИ. Указывать промпты можно в любом месте в файле (например, на второй строке сразу после заголовка), при этом записываются они в той же самой форме, что и в файле `sai_prompts.md`. Одновременно можно указать несколько промптов. Отметим, что имя для промпта можно использовать любое, но важно, чтобы аналогичное имя не возникло случайно в тексте вопроса или ответа
    > По умполчанию используется `sai_prompts_demo.md`. При использовании собственного файла, необходимо прописать соответствующий путь в файле конфигурации `sai_config.toml`
    > Обратите также внимание на скрипт `prompts.py`, в котором можно вручную прописывать логику для сложных промптов. Для таких промптов контент генерируется автоматически скриптом, а в файле `sai_prompts.md` для наглядности указывается `---Автоматический промпт---`

6. Опционально можно уточнить используемые модели и их сокращённые имена в файле `models.py`
    > При запуске данного скрипта (`models.py`), информация (цены запросов на openrouter) в словаре с моделями будет автоматически обновлена, то есть файл будет перезаписан


## Использование

1. Запустить основной скрипт на сервере:
    ```bash
    conda activate sai && clear && python server.py
    ```

2. Запустить скрипт на клиенте для копирования данных с клиента на сервер и логов с сервера:
    ```bash
    conda activate sai && clear && python clone.py
    ```
    > Удобно создать алиас в конфиге терминала (например, `.bashrc`) для быстрого запуска, например: `alias saic="conda activate sai && clear && python /Users/andrei/_/sai/_repo/saibraman/clone.py"` и далее при необходимости просто набирать команду `saic`

3. Запустить скрипт на клиенте для обновления информации о моделях:
    ```bash
    conda activate sai && clear && python models.py
    ```
    > При запуске `clone.py` данный скрипт также автоматически вызывается
    > Удобно создать алиас в конфиге терминала (например, `.bashrc`) для быстрого запуска, например: `alias saim="conda activate sai && clear && python /Users/andrei/_/sai/_repo/saibraman/models.py"` и далее при необходимости просто набирать команду `saim`

4. Запустить основной скрипт на клиенте:
    ```bash
    conda activate sai && clear && python client.py
    ```
    > Удобно создать алиас в конфиге терминала (например, `.bashrc`) для быстрого запуска, например: `alias sai="conda activate sai && clear && python /Users/andrei/_/sai/_repo/saibraman/client.py"` и далее при необходимости просто набирать команду `sai`

5. Теперь в любой дочерней папке папки `root` (см. конфиг) в подпапке `folder_sai` (см. конфиг) можно создать макрадаун файл с произвольным именем, содержащий запрос к LLM-модели. В конце маркдаун файла следует указать ключевое слово `tag_sai` (см. конфиг) и далее дождаться ответа LLM


## Возможности

### Структура markdown-файла запроса

После запуска клиента в любой подпапке `folder_sai` (см. конфиг) создайте `.md` файл, напишите запрос и в конце поставьте тег `tag_sai` (см. конфиг). Клиент
автоматически структурирует файл: добавит заголовок, блоки вопросов/ответов, стоимость и время ответа.

Для продолжения диалога допишите текст в блок нового вопроса и снова
поставьте `tag_sai` (см. конфиг) вместо `tag_sai_done` (см. конфиг) — вся предыдущая переписка уйдёт как история.

### Выбор модели

В тексте запроса укажите три стрелки, пробел и затем сокращенное или полное имя модели из словаря в `models.py` (например, `cl`). По умолчанию используется
`model_default` из конфига.

### Фразы-шаблоны

Любое вхождение имени фразы из `sai_phrases.md` будет заменено на её текст.

### Промпты

Для подстановки системного промпта из `sai_prompts.md` задайте в начале строки три знака равно, пробел и затем имя промпта из `sai_prompts.md`.
Можно последовательно указать несколько промптов. Для автогенерируемых промптов см. `prompts.py`.

### Вставка содержимого файлов

Если в запросе указать в тройных квадратных скобках путь к файлу (txt/md/docx), то в этом месте будет выполнена подстановка содержимого файла. В случае markdown файла можно при необходимости включать только какую-либо секцию (раздел второго уровня) файла, для этого после имени файла ставится пробел, стрелка, пробел и затем указывается имя заголовка. Отметим, что указывается относительный путь файла, при этом поиск происходит последовательно в `~/Downloads`, затем в папке проекта (в котором находится запрос), затем в `root`.

### Вложения PDF

Если в запросе указать в тройных фигурных скобках путь к pdf-файлу, то он будет приложен к запросу (парсится плагином file-parser). Может быть указано несколько файлов в одном запросе. При продолжении диалога старые PDF не переотправляются (используются подготовленные в рамках OpenRouter аннотации файлов) [TODO: пока что переотправляются, но это будет исправлено].

### Генерация и анализ изображений

В тройных фигурных скобках можно указать путь к графическому изображению. Возвращаемые моделью (если это image-модель) сгенерированные картинки будут сохраняться в той же папке, что и маркдаун файл с запросом. При необходимости одновременной генерации нескольких вариантов картинки, например 5, по одному запросу, в запросе стоит указать опцию вида три знака процента, пробел и далее `image-count: 5`. Аналогичным образом можно указать префикс для сохраняемых картинок, например, `image-prefix: neuron`.

### Задание лимита расходов в день

Дневной лимит трат задается в конфиге в поле `money_day_limit`. При его превышении сервер блокирует запросы до ручного перезапуска.

### Ограничение прав пользователей

Возможно указать ограничения для пользователей сервера на доступные модели. Также для каждого пользователя может быть задан базовый системный промпт.

### Логи и синхронизация с сервером

Скрипт `clone.py` (требует настроенного SSH-доступа!) копирует серверный конфиг на сервер, а также забирает с сервера логи.
